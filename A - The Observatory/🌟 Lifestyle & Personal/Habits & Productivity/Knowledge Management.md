# Knowledge Management

How to capture, organize, and find information — whether in notebooks, digital notes, or a full PKM system like this vault.

---

## The Four Ways to Find Notes

Folders are for **organizing**, not necessarily for **finding** anything. They're more for tidiness. The real ways you find notes:

1. **Searching** — Full-text search (Omnisearch, Cmd+Shift+F)
2. **Browsing** — Walking through folder structure or MOCs
3. **Tagging** — Filtering by topic tags
4. **Serendipity** — Stumbling on connections through backlinks and related notes

> In a linked system like Obsidian, finding through connections (backlinks, MOCs) is often faster than finding through folders.

## Note Taking (Analog)

Your choice of pencils, pens, and paper is personal — experiment and find what works.

### During Lectures / Meetings
- Notes will be **scrambled** — that's fine
- Add small annotations so you can decode them later
- Use **arrows** freely to connect ideas
- **Don't cram** too much on one page
- **White space** keeps things readable
- Can look more like a sketch than notes

### Processing Notes Later
**Focus inward first:**
1. Start at the chapter/topic title
2. Find the subheadings
3. Spot the nuances within each section

**Then focus outward:**
1. Find the **logical coherence** between subheadings
2. Then between major headings
3. Then the overall chapter/topic

**Composing clean notes:**
- Split work by section
- Check coherence periodically
- Write cleanly
- Use **colors** as a tool (but don't overdo it — notes, not paintings)

### The Process
1. Get a notebook for live capture (graph paper, blank — whatever works)
2. At home: find structure and logical coherence. Understand the nuances. Sketch outlines on scratch paper. This usually takes multiple passes.
3. Compose final notes. Clean handwriting, selective use of color.

## Note Taking (Digital / PKM)

### Building a Second Brain (Tiago Forte)

The PARA method organizes information by **actionability**, not topic:
- **Projects** — Active efforts with a deadline
- **Areas** — Ongoing responsibilities
- **Resources** — Reference material for future use
- **Archives** — Completed or inactive items

This vault (Constellation) uses the related **ACE method**:
- **Atlas** (Observatory) = Resources + knowledge
- **Calendar** (Signal Relay) = Time-based entries
- **Efforts** (Foundry) = Projects + Areas

> See [[Constellation Guide]] for how this vault implements these ideas.

**Further reading:** [Building a Second Brain: The Illustrated Notes by Maggie Appleton](https://maggieappleton.com/basb) — a beautiful visual summary of the core concepts.

## Principles

- **Capture first, organize later.** Get things out of your head immediately. Sort them when you have time.
- **Link liberally.** Every note is more valuable when connected to other notes. Use `[[wikilinks]]` freely.
- **Progressive summarization.** Don't try to write the perfect note the first time. Start with a rough capture, then bold the important parts, then highlight within the bold, then write a summary at the top.
- **Notes are for thinking, not storage.** If you never revisit a note, it's not serving you. The value is in the retrieval and the connections.
