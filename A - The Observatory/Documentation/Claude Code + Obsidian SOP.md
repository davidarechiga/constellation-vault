# Claude Code + Obsidian SOP

Standard Operating Procedure for the Claude Code integration with the Constellation Obsidian vault. Reference this when setting up a new machine, debugging skills, or adding new commands.

---

## Overview

Claude Code runs as a CLI inside the vault directory (`~/Constellation/`). It reads vault files directly as markdown вЂ” no plugins or REST API needed. Custom slash commands and automated session logging are configured via the `.claude/` folder, which syncs to all machines via Obsidian Git.

**Source:** Adapted from [Claude Code + Obsidian Reddit guide](https://www.reddit.com/r/ClaudeAI/comments/1qr19df/) (clipped 2026-03-31). Key adaptations: no Dataview, no Local REST API, uses ACE folder structure, `.base` files instead of Dataview queries.

## Slash Commands

All commands live in `.claude/commands/` as flat `.md` files. Invoke by typing `/[name]` in Claude Code.

| Command | Purpose |
|---|---|
| `/resume` | Load context at session start вЂ” reads CLAUDE.md + recent Session Log entries |
| `/morning` | Morning briefing + writes time-blocked schedule into today's daily note |
| `/evening` | Plans tomorrow вЂ” fills Prep For Tomorrow + seeds tomorrow's daily note |
| `/weekly` | Weekly review вЂ” synthesizes 7 days of notes, tasks, habits, projects |
| `/organize-inbox` | Sorts loose notes from `C - Signal Relay/Inbox/` into correct ACE locations |
| `/session-persist` | Manual session save вЂ” writes summary to Session Log + promotes learnings to memory |
| `/vault-navigator` | Efficient note finder вЂ” checks `_index.md` files before falling back to Glob/Grep |
| `/therapy-prep` | Individual therapy prep вЂ” scans journals for emotional patterns, writes talking points |
| `/couples-prep` | Couples therapy prep вЂ” relationship moments, appreciations, growth areas |

### Adding a New Command
1. Create `.claude/commands/[name].md`
2. Add frontmatter: `name`, `description`, `version`, `allowed-tools`
3. Write the skill instructions in the body
4. Git will auto-commit and push within 2 minutes

## Session Memory System

### How it works

```
Session start  в†’  /resume  в†’  loads CLAUDE.md + Session Log
During session в†’  conversation with vault context
Session end    в†’  SessionEnd hook fires  в†’  on-stop.sh appends to Session Log
Manual save    в†’  /session-persist  в†’  detailed entry + memory file promotion
```

### Session Log (`C - Signal Relay/Session Log.md`)
Rolling chronological log. Each entry format:
```markdown
### YYYY-MM-DD HH:MM вЂ” [topic]
**Summary:** [1-sentence AI summary]
**Files:** [[Note 1]], [[Note 2]]
```

Auto-entries are generated by `on-stop.sh` using `git diff` + `claude -p` for AI summarization. Manual entries (via `/session-persist`) include decisions, next actions, and memory promotions.

### CLAUDE.md (Permanent Memory)
The vault root `CLAUDE.md` is loaded by `/resume` every session. It contains:
- Vault structure and ACE folder roles
- Core rules (wikilinks-only, no Dataview, TaskNotes system)
- Agent navigation table
- Proactive task help instructions

Do not delete or rename this file.

## Hook Configuration

### `on-stop.sh` вЂ” Session End Logger
Runs automatically when Claude Code exits. Behavior:
1. Runs `git status --short -- "*.md"` to find changed vault files
2. Filters out noise (daily notes, weekly notes, index files)
3. Calls `claude -p` with a `git diff` to generate a 1-sentence AI summary (15s timeout)
4. Appends the summary + file wikilinks to `C - Signal Relay/Session Log.md`
5. Falls back to a plain file list if the AI call fails or times out

### Hook Registration (`.claude/settings.json`)
```json
{
  "hooks": {
    "SessionEnd": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "bash \"$HOME/Constellation/.claude/hooks/on-stop.sh\""
          }
        ]
      }
    ]
  }
}
```

**Important:** Uses `SessionEnd`, not `Stop`. `Stop` fires after every Claude response (expensive/noisy). `SessionEnd` fires once when you exit.

## Sync Setup

| Setting | Value |
|---|---|
| Plugin | Obsidian Git |
| Auto pull | On vault startup |
| Auto commit + push | Every 2 minutes |
| Remote | GitHub (`constellation-vault`, private) |

### What syncs
Everything in `.claude/` **except** `settings.local.json` (gitignored). This means commands, hooks, rules, and `settings.json` propagate to all machines automatically.

### What stays local
`.claude/settings.local.json` вЂ” contains machine-specific permission overrides. Must be configured manually on each machine.

## New Machine Setup

1. Clone the repo: `git clone [repo-url] ~/Constellation`
2. Open vault in Obsidian, install plugins via BRAT if needed
3. Launch Claude Code from vault root: `cd ~/Constellation && claude`
4. Commands and hooks are already present (synced via git)
5. Verify hook works: make a change, exit Claude Code, check `C - Signal Relay/Session Log.md`
6. Configure `.claude/settings.local.json` for any machine-specific permissions

## Troubleshooting

**Skills not appearing as slash commands**
в†’ Check that the `.md` file is in `.claude/commands/`, not `.claude/skills/`. Skills folder is for organization only вЂ” Claude Code reads from `commands/`.

**Session Log not getting entries on exit**
в†’ Verify `settings.json` uses `SessionEnd` (not `Stop`) and the hook path uses `$HOME` not a hardcoded username. Run `bash "$HOME/Constellation/.claude/hooks/on-stop.sh"` manually to test.

**Hook shows "no vault files modified"**
в†’ Normal if no tracked `.md` files were changed. New untracked files won't appear until after first commit. Also fires if you exit without making any changes.

**Merge conflicts in `.obsidian/` files**
в†’ These happen when two machines make different plugin changes. Resolve by manually merging both lists (keep all entries from both sides). Most common in `community-plugins.json`.

**`/resume` shows no session history**
в†’ `Session Log.md` may not exist yet. Run `/session-persist` after a working session to create the first entry, or let the SessionEnd hook fire on next exit.
